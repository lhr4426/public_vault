# 1. 언마운트

- 우선 RAID 구성이랑 실제 폴더 간의 마운트를 제거해야 함

```bash
umount <RAID_구성_위치>
# or
umount <실제_마운트된_폴더>
```

- 예시

```bash
umount /dev/md0
# or
umount /data
```

# 2. LVM 사용 여부 확인 후 제거

- LVM에 대해서는 다음 글에서 좀 더 찾아 볼 예정임
- 지금은 일단 파티션 추상화 정도로 알고있음
- 사실 이번 시리즈에서는 LVM 설정한 적이 없어서 여부 확인을 굳이 안해도 되긴 하는데, 혹시 모르니 해봄
- 실제로 LVM 설정이 되어있던 경우에 갑자기 RAID 구성 빼버리면 오류 날 수도 있다고 함


```bash
pvs
```

- pvs 명령어를 치면 LVM을 사용하는 디스크/파티션들이 보임
- 만약 거기에 RAID 구성했던게 있으면 아래의 명령어를 사용하셈

```bash
vgremove <VG_이름>
pvremove <RAID_구성_위치>
```

# 3. RAID 어레이 사용 중지


```bash
mdadm --stop <RAID_구성_위치>
```

- 예시

```bash
mdadm --stop /dev/md0
```

# 4. RAID 메타데이터 삭제

- 기존 RAID 구성에 포함되어 있던 디스크들은 슈퍼블록(메타데이터)이 남아있을 수 있음
- 그걸 지워야 함

```bash
mdadm --zero-superblock <디스크_위치>
```

- 예시

```bash
mdadm --zero-superblock /dev/sdb
```


# 5. mdadm 설정 파일 수정 및 initramfs 재생성


```bash
vi /etc/mdadm.conf
```

- 위 명령어 치면 ARRAY... 라고 줄이 있을 텐데, 그걸 지움

- 그리고 mdadm 파일에 영향을 받는 initramfs도 재생성 해줘야 함

```bash
dracut -H -f /boot/initramfs-$(uname -r).img $(uname-r)
```


# 6. fstab 수정

```bash
vi /etc/fstab
```

- 원래 마운팅 되어있던 폴더 이름이 포함된 행을 지우면 됨
- 그러고 저장한 다음에 검증


```bash
mount -a
```

- 끝! (불안하면 lsblk 해서 상태 보면 됨)
