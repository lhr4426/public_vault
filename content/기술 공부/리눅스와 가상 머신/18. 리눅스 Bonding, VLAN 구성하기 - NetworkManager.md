# 0. 시작 전 설정

## Bonding 설정

- (당연하지만) 본딩은 NIC가 두 개 이상 있어야지 의미가 있음
- 그러니 네트워크를 추가하기

## VLAN 설정

- (UTM 기준) 네트워크 모드를 브릿지(고급) 로 하지 않으면 L2 헤더를 지워버려서 VLAN 헤더가 날라간다고 함!


# 1. Bonding

## 1) 마스터 NIC 생성


```bash
nmcli con add \
	type bond \
	con-name <이름> \
	ifname <인터페이스_이름> \ 
	mode <모드> \
	miimon <모니터링_주기>
```

- 예시

```bash
nmcli con add \
	type bond \
	con-name bond0 \
	ifname bond0 \ 
	mode active-backup \
	miimon 100
```

- 결과  

![](../../image/스크린샷%202026-01-23%20오후%203.14.04.png)


- 본딩 모드에 대해서는 [이 글](../네트워크/02.%20Bonding.md) 참조
- miimon : Media Independent Interface MONitoring. 슬레이브 NIC를 몇 ms에 한 번 체크할지 결정


## 3) 슬레이브 NIC 생성


```bash
nmcli con add \
	type ethernet \
	slave-type bond \
	con-name <부를_이름> \
	ifname <실제_NIC_이름> \ 
	master <마스터_NIC명>
```

- 예시


```bash
nmcli con add \
	type ethernet \
	slave-type bond \
	con-name slave-enp0s1 \
	ifname enp0s1 \ 
	master bond0
```


```bash
nmcli con add \
	type ethernet \
	slave-type bond \
	con-name slave-enp0s2 \
	ifname enp0s2 \ 
	master bond0
```

- 결과  

![](../../image/스크린샷%202026-01-23%20오후%203.16.01.png)


## 4) 제대로 본딩 됐나 검사


```bash

```


# 2. VLAN

## 1) VLAN 생성


```bash
nmcli con add \
	type vlan \
	con-name <마스터_NIC명>.<VLAN_ID> \ 
	ifname <마스터_NIC명>.<VLAN_ID> \
	vlan.parent <마스터_NIC명> \
	vlan.id <VLAN_ID> \
	ipv4.method manual \
	ipv4.addresses <IP/서브넷마스크>
```

- 예시


```bash
nmcli connection add \ 
	type vlan \
	con-name bond0.10 \
	ifname bond0.10 \
	vlan.parent bond0 \
	vlan.id 10 \
	ipv4.method manual \
	ipv4.addresses 192.168.10.20/24
```

```bash
nmcli connection add \ 
	type vlan \
	con-name bond0.20 \
	ifname bond0.20 \
	vlan.parent bond0 \
	vlan.id 20 \
	ipv4.method manual \
	ipv4.addresses 192.168.20.20/24
```

## 2) VLAN 올리기


```bash
nmcli con up <con-name>
```

- 예시

```bash
nmcli con up bond0.10
```
