# LVM?

- Logical Volume Manager. 저장 공간을 효율적으로 관리하는 패키지
- 저장 공간을 증설하거나 축소할 수 있음

## LVM의 구조

![](../../image/스크린샷%202026-01-19%20오후%2012.59.56.png)    

- PV : Physical Volume. 실제 디스크일 수도 있고, 분할된 파티션일 수도 있음
	- PE : Physical Extent. PV를 구성하는 일정 크기의 블록. 보통 1 PE는 4 MB임
- VG : Volume Group. PV들을 모아서 만든 큰 단위 (스토리지 풀). LV는 이걸 쪼개서 원하는 만큼의 크기의 LV를 만듬
- LV : Logical Volume. 사용자가 실제로 사용하게 되는 단위
	- LE : Logical Extent. LV를 구성하는 일정 크기의 블록인데, PE와 동일한 크기를 가진다.


## RAID와 동시 사용 가능?

- 예스!
- RAID로 구성하게 되면 그것 자체를 하나의 디스크로 보기 때문에, 구성된 RAID 디스크를 LVM을 통해 나눠서 쓸 수 있음.
- 그리고 그것이 오늘의 실습

# 0. 시작 전 상황 설명

- RAID 구성은 현재...  

![](../../image/스크린샷%202026-01-19%20오후%201.10.43.png)    

- RAID 5 레벨로 되어있으며, 세 개의 디스크 (sdb, sdc, sdd)를 사용 중임
- 그리고 RAID 디스크는 /dev/md0 에 위치함
- 마운트 포인트는 /data 임

## 참고

- 디스크/파티션에 파일 시스템이 직접 올라가있는 경우에는 LVM으로 설정이 바로 안됨.
- 아래와 같이 뜸

- 이런 경우에는 언마운트를 한 번 해줘야 함

```bash
umount <디스크_혹은_폴더>
```

# 1. RAID 디스크를 LVM이 인식하도록 PV로 설정


```bash
pvcreate <RAID_구성_위치>
```

- 예시


```bash
pvcreate /dev/md0
```

- 이전 예제를 따라 만들었다면 /dev/md0에 xfs 파일시스템이 설치되어 있을 것임
- 그래서 아래와 같은 WARNING 문구가 뜨는데, 어차피 밀어버릴 거니까 Y 누르고 엔터 하면 됨
- 그럼 pv가 생성됨  

![](../../image/스크린샷%202026-01-19%20오후%201.21.57.png)  


- pvs 명령어로 확인해보면 아래처럼 /dev/md0 이 추가된 모습을 볼 수 있음
![](../../image/스크린샷%202026-01-19%20오후%201.23.40.png)    


## 2. PV로 VG 생성


```bash
vgcreate <VG_이름> <PV_1> <PV_2> ...
```


- 예시


```bash
vgcreate vg_data /dev/md0
```


- 만약에 PE의 기본값(4mb)을 다르게 하고싶다면 아래와 같이 작성

```bash
vgcreate -s <PE_크기> <VG_이름> <PV_1> <PV_2> ...
```

- 예시

```bash
vgcreate -s 8M vg_data /dev/md0
```


![](../../image/스크린샷%202026-01-19%20오후%201.28.52.png)  

## 참고

- RAID로 구성되어있는 PV 하나, 그냥 디스크 하나를 설정한 PV 하나가 있을 때
- 물리적 출처가 다르더라도 둘 다 PV로 설정되어 있다면 vgcreate를 통해서 그룹화할 수 있다
- 다만 권장하지는 않음.


# 3. VG를 사용해 LV 만들기

- 방법이 두 가지 있음.

## 첫 번째 방법 : 크기를 지정해서 생성

```bash
lvcreate -L <크기> -n <LV_이름> <VG_이름>
```

- 예시

```bash
lvcreate -L 24G -n lv_main vg_data
```

- 성공하면 아래와 같이 출력됨

![](../../image/스크린샷%202026-01-19%20오후%201.37.27.png)

## 두 번째 방법 : PE의 갯수를 통해 생성


```bash
lvcreate -l <PE_갯수> -n <LV_이름> <VG_이름>
```

- 근데 이 방법은 가능한 방법의 수가 여러 개 존재함

| PE_갯수 자리에 들어가는 명령어 | 설명                        |
| ------------------ | ------------------------- |
| 숫자                 | 그 숫자 만큼의 PE로 LV을 만듬       |
| n%FREE             | 남아있는 전체 PE만큼의 n%로 LV를 만듬  |
| n%VG               | VG 전체 PE만큼의 n%로 LV를 만듬    |
| n%PVS              | 지정한 PV들의 PE 합의 n%로 LV를 만듬 |

- 예시

```bash
lvcreate -l 100%FREE -n lv_sub vg_data
```


![](../../image/스크린샷%202026-01-19%20오후%201.52.17.png)  


- 두 가지 방법으로 생성하면, lvs 커맨드로 확인했을 때 생성된 두 개의 LV를 볼 수 있음

![](../../image/스크린샷%202026-01-19%20오후%201.52.52.png)  

# 4. 파일 시스템 지정하기

- 이렇게 해서 만든 lv들은 보통 아래의 경로에 저장됨


```bash
/dev/<VG_이름>/<LV_이름>
```

- 예시

```bash
/dev/vg_data/lv_main
/dev/vg_data/lv_sub
```

- 이전에 RAID 구성할 때 파일 시스템 지정한 것과 같이 해주면 됨


```bash
mkfs.<파일_시스템_이름> <경로>
```

- 예시


```bash
mkfs.xfs /dev/vg_data/lv_main
mkfs.xfs /dev/vg_data/lv_sub
```

![](../../image/스크린샷%202026-01-19%20오후%201.56.11.png)  


# 5. 마운트

- RAID 구성 때 처럼 해주면 됨


```bash
mount <디스크명> <경로>
```

- 예시


```bash
mount /dev/vg_data/lv_main /data/main
mount /dev/vg_data/lv_sub /data/sub
```

![](../../image/스크린샷%202026-01-19%20오후%201.57.50.png)  


# 6. 부팅 시 자동 마운트 설정

- 마찬가지로 RAID 구성 때 처럼 해주면 됨
- blkid 구한 다음에 그걸로 fstab 설정하기


```bash
blkid <LV_경로>
```

- 예시


```bash
blkid /dev/vg_data/lv_main
```


![](../../image/Pasted%20image%2020260119140822.png)  

- 구한 블록 아이디 가지고 fstab 수정하기

```bash
vi /etc/fstab
```


![](../../image/스크린샷%202026-01-19%20오후%202.07.11.png)    


- 결과 확인하기


```bash
lsblk
```

- 아래처럼 사용하는 세 개의 디스크(sdb, sdc, sdd) 밑에 RAID 구성 (md0)이 있고, 그 밑에 LVM으로 사용되는 VL들이 보이면 성공임

![](../../image/스크린샷%202026-01-19%20오후%202.06.39.png)

- 재부팅 해서 lsblk 으로 한 번 더 확인했을 때 제대로 마운팅 되어있으면 성공!
