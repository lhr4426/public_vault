# 0. RAID 구성하기 전 점검할 것

## 같은 용량의 디스크를 최소 두 개 이상 (RAID 레벨에 따라 더 필요할 수 있음) 준비하기

- 가상머신 설정에 들어가면 디바이스를 추가할 수 있음
- 가상머신을 끄고 추가한 다음에, 재부팅해서 아래의 명령어를 쳐보기


```bash

```

- 만약에 OS를 설치한 디바이스를 제외하고, 총 세 개의 디바이스를 더 추가했다면 아래와 같이 나올 것

# 1. 디스크/파티션 상태 확인하기

```bash
ls -l /dev/sd*
lsblk
fdisk -l
mount | grep sd
```

- 위 명령어의 의미를 알고싶다면 이전 02번 글 확인

## 체크리스트

- [ ] OS 디스크(**`/dev/sda`**) 아닌지
- [ ] RAID에 사용할 디스크들의 크기가 동일한지
- [ ] 마운트된 파티션 없는지
- [ ] LVM/PV로 사용 중 아닌지

# 2. RAID 구성하기

## 기본 명령어

```bash
mdadm --create /dev/md0 \\
  --level=<RAID_LEVEL> \\
  --raid-devices=<N> \\
  /dev/sdX /dev/sdY [...]
```

- —create /dev/md0 : /dev에 md0이라는 이름으로 구성할 것
- —level= : RAID 레벨 설정
    - 설정 가능한 레벨 : 0, 1, 5, 6, 1+0
- —raid-devices= : RAID로 쓸 디바이스 개수
- /dev/sdX /dev/sdY … : RAID로 쓸 디바이스 지정

## 예시 (RAID 5)

```bash
mdadm --create /dev/md0 \\
  --level=5 --raid-devices=3 \\
  /dev/sdb /dev/sdc /dev/sdd
```

- RAID 5로 디바이스 3개 (sdb, sdc, sdd) 써서 md0이라는 논리 디스크 만들 것

- 그러면 mdadm가 그거에 RAID 메타데이터를 만듬  
![](../../image/스크린샷%202026-01-16%20오후%203.59.33.png)
# 3. RAID 구성 후 확인

## 3-1. 동기화 상태 확인

```bash
cat /proc/mdstat
```

### 출력 예시

![](../../image/스크린샷%202026-01-16%20오후%204.00.26.png)
- raid 5로 구성된 sdd, sdc, sdb가 처음 생성되고 나서 패리티 초기 계산을 진행하고 있음



![](../../image/스크린샷%202026-01-16%20오후%204.03.37.png)
- 초기 계산 끝남

## 3-2. RAID 상세 정보 확인

```bash
mdadm --detail /dev/md0
```

### 출력 예시
![](../../image/스크린샷%202026-01-16%20오후%204.04.15.png)

- `State : clean`
    
    → 정상 (rebuild/resync 없음)
    
- `Active Devices : 2`
    
    → 현재 사용 중 디스크 수
    
- `Failed Devices : 0`
    
    → 장애 디스크 없음
    
- `Spare Devices : 0`
    
    → 대기 디스크 없음
    
- **`degraded` / `recovering` / `resyncing`** 상태면 즉시 원인 확인 필요
    

### 체크리스트

- [ ] `State` 가 **clean** 인가
- [ ] `Failed Devices` 가 **0** 인가
- [ ] 모든 디스크가 **active sync** 상태인가
- [ ] `/proc/mdstat` 와 상태가 일치하는가

# 6. 부팅 후 RAID 자동 인식 설정

## 참고 : 부팅의 과정

```bash
BIOS/UEFI
 → kernel
 → initramfs (초기 루트 환경)
 → 실제 루트 파일시스템
 → systemd
 → /etc/fstab 기반 마운트
```

- 따라서 RAID를 인식하려면 initramfs 에 관련 정보가 있어야 한다.

## mdadm 설정 저장

### 기본 명령어

```bash
mdadm --detail --scan >> /etc/mdadm.conf
```

- mdadm —defail —scan : 현재 시스템에 존재하는 RAID 정보를 스캔함
- > > : 왼쪽의 출력을 오른쪽으로 전달함
    
    - > > /etc/mdadm.conf : 스캔한 RAID 정보를 /etc/mdadm.conf 에 저장함
        

### 예시

![](../../image/스크린샷%202026-01-16%20오후%204.09.51.png)

- 이런식으로 생김

## dracut으로 initramfs 파일 수정

- dracut은 현재 시스템 설정과 /etc 폴더 안에 있는 일부 내용을 가지고 initramfs 이미지를 생성한다.

### 기본 명령어

```bash
dracut [옵션] <initramfs_이미지_경로> <커널_버전>
```

### 예시

```bash
dracut -H -f /boot/initramfs-$(uname -r).img $(uname -r)
```

- -H : Host Only. 현재 시스템에 필요한 드라이버만 포함
- -f : force. 기존 내용 덮어쓰기
- /boot/initramfs-$(uname -r).img $(uname -r)
    - uname -r : 현재 커널 버전 정보 출력
    - $(명령) : 명령 결과를 텍스트로 치환
- 예를 들어 커널 버전이 5.xx… 일 때, 위의 예시 명령은 아래로 치환된다.

```bash
dracut -H -f /boot/initramfs-5.xx....img 5.xx...
```

## fstab (파일 시스템 테이블) 수정

### 파일 시스템 UUID 조회

- 기본 명령어

```bash
blkid /dev/md0
```

- 예시  
![](../../image/스크린샷%202026-01-16%20오후%204.16.07.png)

### /etc/fstab 수정

- 기본 명령어

```bash
vi /etc/fstab
```

- 텍스트 편집기로 fstab 들어가서 밑에다가 위에서 확인한 UUID랑 이것저것 쓰면 됨
    
- 예시
    
![](../../image/스크린샷%202026-01-16%20오후%204.19.46.png)

- 의미

```bash
<장치>        <마운트포인트> <FS> <옵션>     <dump> <fsck>
UUID=...      /data          xfs  defaults   0      0
```

|필드|의미|
|---|---|
|1|장치 (UUID 권장)|
|2|마운트 디렉터리|
|3|파일시스템|
|4|마운트 옵션|
|5|dump 백업 여부|
|6|fsck 검사 순서|

### fstab 검증

- 기본 명령어

```bash
mount -a
```

- -a : fstab 읽어서 그 안에 있는거 전부 마운트함
    - 이 때 오류생기면 fstab 잘못되었다는 거니까 재부팅하면 일남
