# 오늘 학습 키워드 

Git, Github
# 오늘 학습 한 내용을 나만의 언어로 정리하기 

## Git

- Git = VCS(Version Control System) : 버전 관리 도구  

VCS로 Github, Git bash, TortoiseSVN, Git Lab, Unity DevOps 등등 있음  

### VSC를 사용하는 이유

일일히 수정사항을 공유하기 힘들다 보니 나왔음  
구글 드라이브와 유사한 시스템  
**공유가 가능한 원격저장소**

## Github 사용법

### Repository 만들기

1. Repository를 만듬
2. Publish를 함 (구름에 올린다는 느낌으로 위로 올라가있는 아이콘)

### 사람 추가하기

1. Settings
2. Collaborators
3. Add people

### 수정사항 서버에 반영하기

1. 수정을 함
2. commit을 함
3. push를 함

- 작업을 나누어 commit 할 수 있음.

### 수정사항 가져오기

- pull을 누르면 됨

### Checkout

- 특정 커밋을 확인
### Branch

- 새 타임라인을 만드는 느낌
- Checkout과 같이 쓰면 특정 커밋에서 분기할 수 있음

### Cherry-pick

- 특정 커밋을 다른데에 붙일 수 있음


### 충돌 해결하기

```
<<<<<< HEAD
내 수정사항
=================
다른사람 수정사항 (서버에 올라간 내용)
>>>>>> 어쩌구저쩌구
```

- 충돌난 사람이랑 얘기해서 수정 사항을 지우면 됨
- 해결이 다 되면 꺽쇠부분도 지우면 됨

### 커밋을 잘못 했을때

- 히스토리에서 우클릭 후 Undo 하면 커밋 취소 가능
- 이거는 원격 저장소에 올리기 전에만 가능함

- 원격 저장소에 올라간 특정 상태로 돌아가고 싶으면 Revert를 해야함
- 원격 저장소에 올린 상태로 만들기 위해 기존에 있는 내용을 반대로 뒤집어서 커밋하는거임

### 이미 올라간 커밋의 이름을 바꾸고 싶을 때

- Amend Commit 하기
