# 오늘 학습 한 내용을 나만의 언어로 정리하기 

## 다른 팀 아이디어

- 이벤트 버스를 인터페이스로 구현
- 우선순위 큐 사용
- 스프라이트 아틀라스
- 주요 로직을 UGS Cloud Code로 돌림
- BigNumeric 자료형 자체를 만듬
- 오클루전 컬링

## 다른 팀 피드백

- 유니티를 맹신하지 않고 따로 파고든 점이 좋았음
- 상용화/출시를 위해 서버를 붙여서 데이터를 관리하는게 좋았음

## 우리 팀 피드백

- 업무 분담이 잘 되어있었음
- 찾아봐야 하는 기능들이 많이 들어가 있었음. BT, HFSM, UniTask 등등.
- 자료 검색을 잘하신 듯
- 구조가 탄탄하게 되어있음.
- 컨텐츠 확장이 용이해 보임.
- 타겟 플랫폼을 좀 명확하게 하고 했으면 좋았을 것
- 리플렉션의 문제점이 있기 때문에, 다른걸 적용하는게 좋았을 듯.
- 타격감 잘 살렸음. 이펙트, 효과음도 잘 맞음.
- 타이밍이 정말 중요한 게임이라고 느낌. (패링, 섬단)
- 튜토리얼에서 패링/섬단은 학습하기가 어려웠음.
- 진정한 재미를 위해서는 패링/섬단 타이밍을 알 수 있는 중간 학습곡선이 있다면 좋을 듯.
- 문제 파악, 해결하고자 하는 의지가 크게 보여서 좋았음.
